# RepositoryBenchmarks

A benchmarking harness for the repository pattern implementations generated by the `dotnet-sqlite-crud-generator` tool. This class provides a consistent set of CRUD operations against `Product` and `User` entities, enabling performance measurement and comparison across different repository backends or configurations. It encapsulates setup, execution of individual operations, and cleanup within a disposable scope.

## API

### Setup
```csharp
public async Task Setup()
```
Prepares the underlying data store and repository instances for benchmarking. Must be called before any operation methods. Throws if the storage layer cannot be initialized or schema migrations fail.

### GetByIdProductAsync
```csharp
public async Task<Product?> GetByIdProductAsync()
```
Retrieves a `Product` entity by its primary key. Returns the entity if found; otherwise `null`. The key value is determined internally by the benchmark fixture. Throws on database connectivity failures.

### GetByIdUserAsync
```csharp
public async Task<User?> GetByIdUserAsync()
```
Retrieves a `User` entity by its primary key. Returns the entity if found; otherwise `null`. Throws on database connectivity failures.

### GetAllProductsAsync
```csharp
public async Task<IEnumerable<Product>> GetAllProductsAsync()
```
Returns all `Product` entities from the store as an enumerable collection. May return an empty sequence if no records exist. Throws on database connectivity failures.

### GetAllUsersAsync
```csharp
public async Task<IEnumerable<User>> GetAllUsersAsync()
```
Returns all `User` entities from the store as an enumerable collection. May return an empty sequence if no records exist. Throws on database connectivity failures.

### FindProductsAsync
```csharp
public async Task<IEnumerable<Product>> FindProductsAsync()
```
Queries `Product` entities using a predefined filter expression established during setup. Returns matching entities or an empty sequence. Throws on database connectivity failures or if the filter is invalid.

### FindUsersAsync
```csharp
public async Task<IEnumerable<User>> FindUsersAsync()
```
Queries `User` entities using a predefined filter expression established during setup. Returns matching entities or an empty sequence. Throws on database connectivity failures or if the filter is invalid.

### CountProductsAsync
```csharp
public async Task<int> CountProductsAsync()
```
Returns the total number of `Product` records in the store. Throws on database connectivity failures.

### CountUsersAsync
```csharp
public async Task<int> CountUsersAsync()
```
Returns the total number of `User` records in the store. Throws on database connectivity failures.

### ExistsProductAsync
```csharp
public async Task<bool> ExistsProductAsync()
```
Checks whether a specific `Product` record exists by its identifier. Returns `true` if the record is present; `false` otherwise. Throws on database connectivity failures.

### ExistsUserAsync
```csharp
public async Task<bool> ExistsUserAsync()
```
Checks whether a specific `User` record exists by its identifier. Returns `true` if the record is present; `false` otherwise. Throws on database connectivity failures.

### AddProductAsync
```csharp
public async Task<Product> AddProductAsync()
```
Creates a new `Product` entity with benchmark-defined seed data and persists it. Returns the added entity with its assigned identifier populated. Throws if constraints are violated or the store is unavailable.

### AddUserAsync
```csharp
public async Task<User> AddUserAsync()
```
Creates a new `User` entity with benchmark-defined seed data and persists it. Returns the added entity with its assigned identifier populated. Throws if constraints are violated or the store is unavailable.

### UpdateProductAsync
```csharp
public async Task<bool> UpdateProductAsync()
```
Modifies an existing `Product` entity with predetermined changes and persists them. Returns `true` if the update succeeded; `false` if the target record was not found. Throws on database connectivity failures or constraint violations.

### UpdateUserAsync
```csharp
public async Task<bool> UpdateUserAsync()
```
Modifies an existing `User` entity with predetermined changes and persists them. Returns `true` if the update succeeded; `false` if the target record was not found. Throws on database connectivity failures or constraint violations.

### DeleteProductAsync
```csharp
public async Task<bool> DeleteProductAsync()
```
Removes a specific `Product` entity from the store. Returns `true` if the deletion succeeded; `false` if the record was not found. Throws on database connectivity failures.

### DeleteUserAsync
```csharp
public async Task<bool> DeleteUserAsync()
```
Removes a specific `User` entity from the store. Returns `true` if the deletion succeeded; `false` if the record was not found. Throws on database connectivity failures.

### SaveChangesAsync
```csharp
public async Task<int> SaveChangesAsync()
```
Persists all pending changes tracked by the underlying unit-of-work or change tracker. Returns the number of state entries written to the store. Throws if the save operation fails due to concurrency conflicts or connectivity issues.

### Cleanup
```csharp
public async Task Cleanup()
```
Resets the data store to a clean state by removing benchmark-generated records and releasing transient resources. Should be invoked between benchmark iterations to maintain isolation. Throws if the cleanup operations cannot be completed.

### Dispose
```csharp
public void Dispose()
```
Releases all managed and unmanaged resources held by the benchmark harness, including database connections and repository instances. Safe to call multiple times.

## Usage

### Example 1: Single-iteration benchmark with BenchmarkDotNet
```csharp
[Benchmark]
public async Task<Product> Benchmark_AddProduct()
{
    await _benchmarks.Setup();
    var result = await _benchmarks.AddProductAsync();
    await _benchmarks.Cleanup();
    return result;
}
```

### Example 2: Manual loop measuring multiple operations
```csharp
using var benchmarks = new RepositoryBenchmarks();
await benchmarks.Setup();

var stopwatch = Stopwatch.StartNew();
for (int i = 0; i < 1000; i++)
{
    await benchmarks.AddUserAsync();
}
stopwatch.Stop();
Console.WriteLine($"1000 inserts: {stopwatch.ElapsedMilliseconds} ms");

await benchmarks.Cleanup();
```

## Notes

- **Setup and Cleanup pairing**: `Setup` must be called before any operation method. `Cleanup` should be called after each benchmark iteration to prevent accumulated state from skewing subsequent measurements. Failing to call `Cleanup` between iterations may cause primary-key collisions on `Add*` methods.
- **Dispose semantics**: Once `Dispose` is invoked, all further calls to operation methods will fail with `ObjectDisposedException` or equivalent. The class does not support re-initialization after disposal; create a new instance for each benchmark session.
- **Thread safety**: This class is not thread-safe. All methods are designed for sequential invocation from a single thread. Concurrent calls to `Setup`, operation methods, or `Cleanup` may corrupt internal state or cause race conditions in the underlying SQLite store.
- **Return value conventions**: `GetById*` methods return `null` when the requested entity does not exist. `Update*` and `Delete*` methods return `false` when the target record is absent, rather than throwing. This allows benchmarks to measure both the happy path and the not-found path without exception overhead.
- **Filtered queries**: `FindProductsAsync` and `FindUsersAsync` rely on filter criteria configured during `Setup`. If `Setup` has not been called, or if the filter references invalid fields, these methods will throw.
- **SaveChangesAsync independence**: This method flushes the change tracker independently of the individual operation methods, which may or may not auto-save depending on the repository implementation under test. Benchmarks should account for this when measuring unit-of-work patterns.
