# DbContextProvider
Provides access to the underlying EF Core `DbContext` and repository instances generated by the SQLite CRUD generator, handling transaction management and resource disposal.

## API
### DbContextProvider()
Initializes a new instance of the `DbContextProvider` class. Sets up an internal `DbContext` based on the application's configuration.

### public async Task<int> SaveChangesAsync()
Asynchronously persists all changes made in this context to the database.  
**Return value:** Number of state entries written to the database.  
**Throws:**  
- `DbUpdateException` if the save operation fails due to a database error.  
- `ObjectDisposedException` if the provider has been disposed.

### public async Task BeginTransactionAsync()
Begins a new database transaction asynchronously.  
**Throws:**  
- `InvalidOperationException` if a transaction is already active.  
- `ObjectDisposedException` if the provider has been disposed.

### public async Task CommitTransactionAsync()
Commits the current transaction asynchronously.  
**Throws:**  
- `InvalidOperationException` if no transaction has been begun.  
- `ObjectDisposedException` if the provider has been disposed.

### public async Task RollbackTransactionAsync()
Rolls back the current transaction asynchronously.  
**Throws:**  
- `InvalidOperationException` if no transaction has been begun.  
- `ObjectDisposedException` if the provider has been disposed.

### public void Dispose()
Releases all resources used by the `DbContextProvider` synchronously.  
**Throws:** No exceptions under normal use; calling `Dispose` multiple times is safe.

### public async ValueTask DisposeAsync()
Asynchronously releases all resources used by the `DbContextProvider`.  
**Throws:** No exceptions under normal use; calling `DisposeAsync` multiple times is safe.

## Usage
```csharp
using var ctx = new DbContextProvider();

// Begin a transaction, make changes, and commit.
await ctx.BeginTransactionAsync();
var userRepo = new UserRepository(ctx.DatabaseConnection);
var newUser = new User { Email = "alice@example.com", Username = "alice" };
await userRepo.AddAsync(newUser);
int saved = await ctx.SaveChangesAsync(); // persisted within transaction
await ctx.CommitTransactionAsync();
```
```csharp
await using var ctxAsync = new DbContextProvider();
var productRepo = new ProductRepository(ctxAsync.DatabaseConnection);
var lowStock = await productRepo.GetLowStockAsync(10);
foreach (var p in lowStock)
{
    Console.WriteLine($"{p.Sku}: {p.Name} stock={p.Stock}");
}
```
## Notes
- The `DbContextProvider` is **not thread‑safe**; concurrent calls to `SaveChangesAsync` or transaction methods from multiple threads may result in undefined behavior.  
- Calling `Dispose` or `DisposeAsync` more than once is safe; subsequent calls are no‑ops.  
- Transaction methods must be used in the correct order (`BeginTransactionAsync` → … → `CommitTransactionAsync` or `RollbackTransactionAsync`); attempting to commit or roll back without an active transaction throws `InvalidOperationException`.  
- Repository constructors do not take ownership of the supplied `DatabaseConnection`; the connection's lifetime must exceed that of the repository.  
- All asynchronous query methods return `null` or empty enumerables when no matching records are found; they never throw for missing data.  
- Exceptions originating from the underlying EF Core provider (e.g., `DbUpdateException`) are propagated unchanged.
