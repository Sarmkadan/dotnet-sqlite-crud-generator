# QueryBuilderGenerationServiceTests

The `QueryBuilderGenerationServiceTests` class serves as the comprehensive test suite for validating the logic within the query builder generation service of the `dotnet-sqlite-crud-generator` project. It verifies that the generated C# source code correctly incorporates entity metadata, such as class names, table names, and property-specific filtering logic, while ensuring robust error handling for null inputs and successful file system operations during asynchronous generation tasks.

## API

### Properties

*   **`public int Id`**
    Represents a unique identifier used within the test context, typically for mock data setup or assertion verification.

*   **`public string Name`**
    Holds a string value representing a name field, utilized in test scenarios to validate string property handling in generated queries.

*   **`public decimal Price`**
    Contains a decimal value representing a price field, used to verify that numeric properties are correctly typed and handled in the generated query builder.

*   **`public bool IsActive`**
    Stores a boolean flag indicating an active state, employed to test boolean property inclusion in strongly typed where clauses.

*   **`public QueryBuilderGenerationServiceTests`**
    A public member of the type itself, potentially used for self-referential test scenarios or specific framework requirements within the test harness.

### Methods

*   **`public void Dispose()`**
    Releases unmanaged resources and performs cleanup operations associated with the test instance. This method does not return a value and should be called when the test fixture is torn down.

*   **`public void BuildQueryBuilderSource_ContainsClassName()`**
    Asserts that the source code generated by the builder includes the correct class name of the target entity. It takes no parameters and returns void; it fails the test if the class name is missing or incorrect.

*   **`public void BuildQueryBuilderSource_ContainsTableName()`**
    Verifies that the generated source code explicitly defines the correct database table name associated with the entity. No parameters are accepted, and it returns void.

*   **`public void BuildQueryBuilderSource_ContainsStronglyTypedWhereForEachProperty()`**
    Ensures that the generated query builder includes strongly typed `Where` method implementations for every property defined in the entity (e.g., `Id`, `Name`, `Price`, `IsActive`). It accepts no parameters and returns void.

*   **`public void BuildQueryBuilderSource_ContainsBuildMethod()`**
    Checks for the presence of a `Build` method within the generated source code, which is responsible for constructing the final query object. It takes no parameters and returns void.

*   **`public void BuildQueryBuilderSource_ContainsLimitAndOffset()`**
    Validates that the generated code includes support for pagination via `Limit` and `Offset` methods. No parameters are required, and it returns void.

*   **`public void BuildQueryBuilderSource_ContainsOrderByMethods()`**
    Asserts that the generated source includes methods for ordering results. It takes no parameters and returns void.

*   **`public void BuildQueryBuilderSource_ContainsResetMethod()`**
    Verifies the existence of a `Reset` method in the generated builder to clear current query state. It accepts no parameters and returns void.

*   **`public void BuildQueryBuilderSource_ThrowsOnNullEntityType()`**
    Confirms that the source generation logic throws an appropriate exception when the provided entity type is null. It takes no parameters and returns void; the test fails if no exception is thrown.

*   **`public async Task GenerateQueryBuilderAsync_WritesFileToOutputPath()`**
    Asynchronously verifies that the generation service successfully writes the generated source code to the specified output file path. It returns a `Task` that completes when the verification is done.

*   **`public async Task GenerateQueryBuilderAsync_FileContainsValidSource()`**
    Asynchronously checks that the file written by the generation service contains syntactically valid C# source code. It returns a `Task`.

*   **`public async Task GenerateQueryBuilderAsync_ThrowsOnNullEntityType()`**
    Asynchronously ensures that the generation service throws an exception when invoked with a null entity type. It returns a `Task` and fails if the expected exception does not occur.

## Usage

### Example 1: Verifying Generated Source Content
This example demonstrates how the test suite validates that the generated builder includes specific methods for entity properties.

```csharp
using Xunit;
using System;

public class ValidationScenario
{
    [Fact]
    public void ValidateBuilderStructure()
    {
        var testInstance = new QueryBuilderGenerationServiceTests();
        
        // Execute the test method that asserts the presence of strongly typed Where clauses
        // for properties like Id, Name, Price, and IsActive.
        testInstance.BuildQueryBuilderSource_ContainsStronglyTypedWhereForEachProperty();
        
        // If the method completes without throwing an assertion exception, 
        // the generated source correctly handles all entity properties.
    }
}
```

## Notes

*   **Thread Safety**: As is standard for xUnit test classes, instances of `QueryBuilderGenerationServiceTests` are not designed to be thread-safe across concurrent modifications of their internal state during execution. Each test method should ideally operate on a fresh instance or ensure isolation.
*   **Resource Management**: The presence of the `Dispose` method indicates that the test class may allocate unmanaged resources or hold file handles during execution, particularly during the `GenerateQueryBuilderAsync` tests. Consumers extending or modifying these tests must ensure `Dispose` is called during teardown to prevent resource leaks.
*   **Null Handling**: Multiple methods (`BuildQueryBuilderSource_ThrowsOnNullEntityType`, `GenerateQueryBuilderAsync_ThrowsOnNullEntityType`) explicitly validate null argument enforcement. Implementations relying on this service must not bypass these checks, as the generation logic is designed to fail fast on null entity types.
*   **File System Dependencies**: The asynchronous methods imply a dependency on the file system. Tests may fail in environments with restricted write permissions or invalid output paths, which should be considered when running in constrained CI/CD pipelines.
